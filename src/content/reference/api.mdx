# REST API

Martis exposes a fully documented REST API for every resource, metric, action, lens, filter, and dashboard. The schema is generated at runtime by [Scramble](https://scramble.dedoc.co/) (OpenAPI 3.1) and served alongside the panel.

## Live OpenAPI

When Martis is installed in your application, the live, browsable Swagger UI is at:

```
GET /martis/api-docs
```

The raw OpenAPI document is at:

```
GET /martis/api-docs.json
```

Both routes are protected by the same auth guard as the rest of Martis. They are intended for in-team consumption — if you need to expose them publicly, gate them behind your own middleware.

## Base URL

| Item     | Value                                                          |
|----------|----------------------------------------------------------------|
| Base     | `/martis/api`                                                  |
| Format   | `application/json`                                             |
| Auth     | Sanctum Bearer token (configurable via `MARTIS_GUARD`)         |
| CSRF     | Required for cookie-authenticated calls; not required for Bearer |

## Authentication

### Login

```http
POST /martis/login
Content-Type: application/json

{ "email": "admin@example.com", "password": "..." }
```

Response:

```json
{ "token": "1|abc123..." }
```

### Using the token

Include it on every subsequent request:

```http
Authorization: Bearer 1|abc123...
```

### Current user

```http
GET /martis/api/auth/user
```

### Logout

```http
POST /martis/api/auth/logout
```

## Resource endpoints

Every registered resource exposes a stable family of endpoints, derived from its `uriKey`:

| Method | Path                                       | Purpose                                  |
|--------|--------------------------------------------|------------------------------------------|
| GET    | `/martis/api/resources`                    | List all registered resources            |
| GET    | `/martis/api/resources/{uriKey}/index`     | Catalog rows + total                     |
| GET    | `/martis/api/resources/{uriKey}/{id}`      | Single resource detail                   |
| POST   | `/martis/api/resources/{uriKey}`           | Create                                   |
| PUT    | `/martis/api/resources/{uriKey}/{id}`      | Update                                   |
| DELETE | `/martis/api/resources/{uriKey}/{id}`      | Delete                                   |
| GET    | `/martis/api/resources/{uriKey}/lenses`    | Available lenses for this resource       |
| POST   | `/martis/api/resources/{uriKey}/actions`   | Run a resource or bulk action            |

The exact request/response shapes are documented in the live OpenAPI spec — read it from the running app rather than copying examples.

## Metrics

```http
GET /martis/api/resources/{uriKey}/metrics/{metricUriKey}?range=30
```

The `range` query parameter is optional and defaults to the metric's first declared range. Responses always include `value`, `previous`, `format`, and `prefix`/`suffix` keys when applicable.

## Actions

```http
POST /martis/api/resources/{uriKey}/actions/{actionUriKey}
Content-Type: application/json

{
  "models": [1, 2, 3],
  "fields": { "cc": "billing@example.com" }
}
```

Bulk actions accept an array of model keys. Standalone actions accept an empty `models` array. Queued actions return immediately with a `202` and a job id.

## Filters and lenses

Filters and lenses are applied via the catalog query string:

```
GET /martis/api/resources/clients/index?perPage=25&filters[status]=active&lens=churn-risk
```

Each registered filter declares its own URI key and value shape; the OpenAPI spec exposes them as named parameters per resource.

## Pagination

Catalog responses always include:

```json
{
  "data":  [...],
  "total": 1234,
  "perPage": 25,
  "currentPage": 1
}
```

`perPage` is bounded by `martis.catalog.max_per_page` (default 100). Requests over the cap are clamped silently.

## Error envelope

All error responses follow:

```json
{
  "message": "Validation failed.",
  "errors":  { "email": ["The email field is required."] }
}
```

Status codes follow Laravel conventions: `422` for validation, `403` for policy denial, `404` for missing rows, `500` for unhandled. The frontend's `ResourceErrorPage` triages by status — see [v1.1.1 release notes](https://github.com/Real-Edge-FX/martis-package/releases/tag/v1.1.1).

## Custom endpoints

If you need to extend the API, add routes inside a Tool (`Tool::routes()`) or via your own service provider. See [Custom Tools](/docs/customization/tools) for the lifecycle hooks.
